# Headmap & Labelmap Visualization

---

## What is this folder?

This folder contains PNG images generated from the `.npz` files in `head_base_frame_npz/`. Those `.npz` files are the final preprocessed data that the CMMST model reads during training and evaluation. Visualizing them here lets you confirm that the preprocessing pipeline (quaternion → viewport → tile maps) produced correct results.

---

## Output structure

```
headmap_labelmap_png/
└── topic0_user1/
    ├── trajectory_headmap.png      <- overview: accumulated head attention over all frames
    ├── trajectory_labelmap.png     <- overview: tile visit frequency on 9x16 grid
    ├── trajectory_path.png         <- overview: full head movement path (blue=start, red=end)
    ├── headmaps/
    │   ├── frame_00000.png         <- individual headmap frame
    │   ├── frame_00081.png
    │   └── ...
    └── labelmaps/
        ├── frame_00000.png         <- individual labelmap frame
        ├── frame_00081.png
        └── ...
```

## Understanding the images

### trajectory_path.png
The most informative image for verifying correctness. Shows the full viewport movement path drawn on the equirectangular space (144x256 pixels, upscaled 4x):
- **Blue line** = beginning of the session
- **Red line** = end of the session
- **Green dot** = first frame position
- **Red square** = last frame position

### trajectory_headmap.png
All headmap frames summed together and normalized. Bright regions (white/yellow) = areas the user looked at most. Dark/black = areas never visited. Shows the overall attention distribution across the whole video session.

### trajectory_labelmap.png
Same idea but at tile level. The 9x16 tile grid is shown with a hot colormap (black -> red -> yellow -> white). Brighter tiles = more frames spent looking there. Useful to verify that the tile grid aligns with the headmap positions.

### headmaps/frame_NNNNN.png
One PNG per sampled frame. Each image is 1024x576 pixels (256x144 upscaled 4x). What you see:
- **Black background** = no head activity (stored as -1 in the .npz, clipped to 0 for display)
- **Small bright blob** = where the user was looking (gaussian kernel, ~5x5 pixels at original resolution)
- Colormap: black -> red -> yellow -> white (hot)

The blob is intentionally small — the raw headmap stores a narrow gaussian centered on the exact head position, not a smoothed region.

### labelmaps/frame_NNNNN.png
One PNG per sampled frame. Shows the 9x16 tile grid (384x216 pixels, each tile = 24x24 pixels):
- **White tile** = the tile the user's viewport is currently centered on (1 active tile per frame)
- **Black tiles** = all other tiles
- **Dark gray lines** = tile grid boundaries

## Data source

| .npz key | Shape | Dtype | Range | Meaning |
|----------|-------|-------|-------|---------|
| `headmaps` | (N_frames, 144, 256) | float16 | -1 to 1 | Gaussian blob at head position per frame |
| `labelmaps` | (N_frames, 144) | uint8 | 0 or 1 | One-hot tile index (9x16=144 tiles) per frame |

The `.npz` files are located in `head_base_frame_npz/topic{T}/data_topic{T}_user{U}.npz`.
They are generated by `CMMST/CMMST-main/tools/get_act_tile_base_frame_parallel.py` from the viewport pickle files produced by `CMMST/CMMST-main/PanoSaliency/get_viewport.py`.
